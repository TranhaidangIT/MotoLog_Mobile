# MotoLog – V7: Màn hình "Thiết lập ban đầu" cho Bảo dưỡng

> Sửa lỗi: hạng mục nào có `lastDoneOdo = 0` (mặc định) sẽ luôn báo "Còn 0 km" ngay khi ODO xe > chu kỳ, dù thực tế người dùng có thể đã thay gần đây. Thêm bước hỏi 1 lần khi thêm xe mới, để mỗi hạng mục có điểm khởi đầu đúng.

---

## 1. Khi nào màn hình này xuất hiện

- **Lần đầu thêm xe mới** (ngay sau khi điền xong thông tin xe ở `MyVehicleScreen`/màn "Thêm xe") → tự động chuyển sang màn này trước khi vào Home.
- Cũng cho phép vào lại sau này qua: `MyVehicleScreen` → menu (icon 3 chấm hoặc trong phần Cá nhân) → "Thiết lập lại mốc bảo dưỡng".

---

## 2. Nguyên tắc UX — không bắt người dùng nhớ số chính xác

Với mỗi hạng mục, người dùng có 3 lựa chọn nhanh, không cần nhập tay nếu không nhớ:

| Lựa chọn | Ý nghĩa | `lastDoneOdo` được gán |
|---|---|---|
| **"Vừa thay gần đây"** | Coi như mới thay xong, bắt đầu đếm lại từ đầu | `= currentOdo` (đầy 100% chu kỳ còn lại) |
| **"Chưa từng thay / Không nhớ"** | An toàn nhất — giả định đã dùng hết, nên nhắc luôn | `= currentOdo − intervalKm` (còn lại = 0, hiện cảnh báo ngay) |
| **"Tôi nhớ chính xác ODO lúc thay"** | Người dùng có hoá đơn/ghi chép | Nhập số ODO thật vào ô input |

→ Mặc định khi mới mở màn hình: tất cả hạng mục ở trạng thái **"Vừa thay gần đây"** (lựa chọn lạc quan nhất, tránh doạ người dùng bằng một loạt cảnh báo đỏ ngay khi vừa mở app lần đầu). Người dùng có thể đổi từng hạng mục nếu nhớ rõ hơn.

---

## 3. Bố cục màn hình

AppBar: "Thiết lập mốc bảo dưỡng" (không có nút back nếu là onboarding lần đầu — chỉ có nút "Bỏ qua" góc phải).

Phụ đề ngay dưới AppBar: *"Cho biết lần gần nhất bạn thay từng hạng mục, để app tính đúng số km còn lại"*.

Mỗi hạng mục là 1 card:
- Icon + tên hạng mục (giống style card Bảo dưỡng đã có)
- 3 nút chọn nhanh dạng chip ngang: `Vừa thay` · `Chưa từng / Không nhớ` · `Nhập ODO`
- Nếu chọn "Nhập ODO" → hiện 1 ô input nhỏ ngay dưới, placeholder = "Nhập số km lúc thay"

Cuối màn hình: nút "Hoàn tất" full-width màu xanh đậm.

## 4. Code — `lib/screens/maintenance_setup_screen.dart`

```dart
import 'package:flutter/material.dart';
import 'package:google_fonts/google_fonts.dart';
import '../theme/app_theme.dart';
import '../data/maintenance_store.dart';
import '../models/maintenance_item.dart';

enum _SetupChoice { justDone, neverDone, custom }

class MaintenanceSetupScreen extends StatefulWidget {
  final bool isOnboarding; // true = lần đầu thêm xe, ẩn nút back, hiện nút "Bỏ qua"
  const MaintenanceSetupScreen({super.key, this.isOnboarding = true});

  @override
  State<MaintenanceSetupScreen> createState() => _MaintenanceSetupScreenState();
}

class _MaintenanceSetupScreenState extends State<MaintenanceSetupScreen> {
  final Map<String, _SetupChoice> _choices = {};
  final Map<String, TextEditingController> _customCtrls = {};

  @override
  void initState() {
    super.initState();
    for (final item in MaintenanceStore.instance.items) {
      _choices[item.id] = _SetupChoice.justDone; // mặc định lạc quan
      _customCtrls[item.id] = TextEditingController();
    }
  }

  void _save() {
    final store = MaintenanceStore.instance;
    final currentOdo = store.currentOdo;

    for (final item in store.items) {
      final choice = _choices[item.id]!;
      int newLastDoneOdo;
      switch (choice) {
        case _SetupChoice.justDone:
          newLastDoneOdo = currentOdo;
          break;
        case _SetupChoice.neverDone:
          newLastDoneOdo = currentOdo - item.intervalKm;
          break;
        case _SetupChoice.custom:
          final parsed = int.tryParse(_customCtrls[item.id]!.text);
          newLastDoneOdo = parsed ?? currentOdo;
          break;
      }
      store.markDone(item.id, newLastDoneOdo);
    }

    Navigator.of(context).pushReplacementNamed('/home'); // hoặc Navigator.pop nếu vào từ menu sau này
  }

  Widget _choiceChip(String itemId, _SetupChoice value, String label) {
    final active = _choices[itemId] == value;
    return GestureDetector(
      onTap: () => setState(() => _choices[itemId] = value),
      child: Container(
        margin: const EdgeInsets.only(right: 6),
        padding: const EdgeInsets.symmetric(horizontal: 10, vertical: 7),
        decoration: BoxDecoration(
          color: active ? AppColors.primary : Colors.white,
          borderRadius: BorderRadius.circular(16),
          border: Border.all(color: active ? AppColors.primary : AppColors.divider),
        ),
        child: Text(label, style: GoogleFonts.beVietnamPro(
          fontSize: 11, fontWeight: FontWeight.w600,
          color: active ? Colors.white : AppColors.textSecondary,
        )),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    final store = MaintenanceStore.instance;

    return Scaffold(
      appBar: AppBar(
        title: const Text('Thiết lập mốc bảo dưỡng'),
        automaticallyImplyLeading: !widget.isOnboarding,
        actions: widget.isOnboarding
          ? [TextButton(onPressed: _save, child: const Text('Bỏ qua'))]
          : null,
      ),
      body: Column(children: [
        Container(
          width: double.infinity,
          color: Colors.white,
          padding: const EdgeInsets.fromLTRB(16, 4, 16, 14),
          child: Text(
            'Cho biết lần gần nhất bạn thay từng hạng mục, để app tính đúng số km còn lại',
            style: GoogleFonts.beVietnamPro(fontSize: 12, color: AppColors.textSecondary),
          ),
        ),
        Expanded(
          child: ListView.separated(
            padding: const EdgeInsets.all(14),
            itemCount: store.items.length,
            separatorBuilder: (_, __) => const SizedBox(height: 10),
            itemBuilder: (_, i) {
              final item = store.items[i];
              final choice = _choices[item.id]!;
              return Container(
                padding: const EdgeInsets.all(14),
                decoration: BoxDecoration(
                  color: Colors.white,
                  borderRadius: BorderRadius.circular(14),
                  boxShadow: [BoxShadow(color: Colors.black.withOpacity(0.04), blurRadius: 6)],
                ),
                child: Column(crossAxisAlignment: CrossAxisAlignment.start, children: [
                  Row(children: [
                    Container(
                      width: 38, height: 38,
                      decoration: const BoxDecoration(color: AppColors.greenChip, shape: BoxShape.circle),
                      child: Icon(item.icon, color: AppColors.primary, size: 20),
                    ),
                    const SizedBox(width: 10),
                    Expanded(child: Text(item.name, style: GoogleFonts.beVietnamPro(fontSize: 14, fontWeight: FontWeight.w600))),
                  ]),
                  const SizedBox(height: 10),
                  Wrap(spacing: 0, runSpacing: 6, children: [
                    _choiceChip(item.id, _SetupChoice.justDone, 'Vừa thay'),
                    _choiceChip(item.id, _SetupChoice.neverDone, 'Chưa từng / Không nhớ'),
                    _choiceChip(item.id, _SetupChoice.custom, 'Nhập ODO'),
                  ]),
                  if (choice == _SetupChoice.custom) ...[
                    const SizedBox(height: 10),
                    TextField(
                      controller: _customCtrls[item.id],
                      keyboardType: TextInputType.number,
                      decoration: InputDecoration(
                        hintText: 'Nhập số km lúc thay',
                        hintStyle: GoogleFonts.beVietnamPro(fontSize: 12, color: AppColors.textSecondary),
                        filled: true,
                        fillColor: const Color(0xFFFAFAFA),
                        contentPadding: const EdgeInsets.symmetric(horizontal: 12, vertical: 10),
                        border: OutlineInputBorder(borderRadius: BorderRadius.circular(8), borderSide: BorderSide(color: AppColors.divider)),
                        enabledBorder: OutlineInputBorder(borderRadius: BorderRadius.circular(8), borderSide: BorderSide(color: AppColors.divider)),
                      ),
                    ),
                  ],
                ]),
              );
            },
          ),
        ),
        Padding(
          padding: const EdgeInsets.all(16),
          child: SizedBox(
            width: double.infinity,
            child: ElevatedButton(
              onPressed: _save,
              style: ElevatedButton.styleFrom(
                backgroundColor: AppColors.primary,
                padding: const EdgeInsets.symmetric(vertical: 14),
                shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(12)),
              ),
              child: Text('Hoàn tất', style: GoogleFonts.beVietnamPro(fontSize: 15, fontWeight: FontWeight.w600, color: Colors.white)),
            ),
          ),
        ),
      ]),
    );
  }
}
```

## 7. Sửa logic bộ đếm — bỏ giới hạn về 0, hiện rõ "Đã quá hạn"

> Bổ sung quan trọng: logic cũ ở `MaintenanceItem` (file V6) dùng `clamp` ép số "còn lại" không bao giờ xuống dưới 0 — khiến hạng mục đã trễ rất lâu vẫn chỉ hiện "Còn 0 km", làm người dùng hiểu lầm là "vừa mới tới hạn" trong khi thực tế có thể đã trễ hàng ngàn km. Sửa lại để phân biệt rõ 2 trạng thái: **chưa tới hạn** (hiện "Còn X km") và **đã quá hạn** (hiện "Đã quá hạn X km", chữ đỏ).

### 7.1 Sửa `MaintenanceItem` — thay thế toàn bộ 2 hàm `remainingKm` và `progress`, thêm `isOverdue`/`overdueKm`

```dart
// Số km còn lại tới hạn — có thể ÂM nếu đã quá hạn (không clamp về 0 nữa)
int kmUntilDue(int currentOdo) {
  final used = currentOdo - lastDoneOdo;
  return intervalKm - used; // âm = đã quá hạn bấy nhiêu km
}

bool isOverdue(int currentOdo) => kmUntilDue(currentOdo) < 0;

// Dùng khi CHƯA quá hạn — số km còn lại để hiện "Còn X km"
int remainingKm(int currentOdo) {
  final v = kmUntilDue(currentOdo);
  return v < 0 ? 0 : v;
}

// Dùng khi ĐÃ quá hạn — số km đã trễ để hiện "Đã quá hạn X km"
int overdueKm(int currentOdo) {
  final v = kmUntilDue(currentOdo);
  return v < 0 ? -v : 0;
}

// progress vẫn clamp 0–1 CHỈ để vẽ progress bar (UI không thể vẽ thanh >100%)
double progress(int currentOdo) {
  final used = currentOdo - lastDoneOdo;
  final p = used / intervalKm;
  return p.clamp(0.0, 1.0);
}

// urgency giữ nguyên logic %, nhưng quá hạn luôn là 'urgent'
String urgency(int currentOdo) {
  if (isOverdue(currentOdo)) return 'urgent';
  final p = progress(currentOdo);
  if (p >= 0.95) return 'urgent';
  if (p >= 0.85) return 'soon';
  return 'normal';
}
```

### 7.2 Sửa UI hiển thị ở `MaintenanceScreen` và `ReminderScreen`

Mọi nơi đang hiện text `'Còn ${item.remainingKm(currentOdo)} km'`, đổi thành kiểm tra `isOverdue` trước:

```dart
Widget _remainingText(MaintenanceItem item, int currentOdo) {
  if (item.isOverdue(currentOdo)) {
    return Text(
      'Đã quá hạn ${item.overdueKm(currentOdo)} km',
      style: GoogleFonts.beVietnamPro(fontSize: 12, color: const Color(0xFFD32F2F), fontWeight: FontWeight.w600),
    );
  }
  return Text(
    'Còn ${item.remainingKm(currentOdo)} km',
    style: GoogleFonts.beVietnamPro(fontSize: 12, color: AppColors.primary, fontWeight: FontWeight.w600),
  );
}
```

Kết quả với ví dụ "Thay nhớt máy" (chu kỳ 2.000, ODO hiện tại 12.354, `lastDoneOdo = 0` vì chưa thiết lập): `kmUntilDue = 2000 - 12354 = -10354` → hiện đúng **"Đã quá hạn 10.354 km"** màu đỏ, thay vì gây hiểu lầm "Còn 0 km".

> Đây cũng là lý do càng cần màn `MaintenanceSetupScreen` ở mục 1–6 phía trên — nếu người dùng thiết lập đúng "Vừa thay gần đây" cho từng hạng mục khi mới thêm xe, sẽ tránh được tình trạng quá hạn giả này ngay từ đầu.

### 7.3 Giải thích ý nghĩa mục "Định kỳ" trong `ReminderScreen` — thêm dòng phụ đề

Để người dùng không thắc mắc "mục Định kỳ là sao", thêm 1 dòng giải thích nhỏ ngay dưới tiêu đề "ĐỊNH KỲ" trong `reminder_screen.dart`:

```dart
Padding(

    style: GoogleFonts.beVietnamPro(fontSize: 11, color: AppColors.textSecondary),
  ),

```

Quy tắc phân loại (không đổi so với V6, chỉ làm rõ lại để AI code không nhầm):

| Điều kiện | Mục hiển thị | Badge |
|---|---|---|
| Đã quá hạn (`isOverdue = true`) hoặc dùng ≥ 95% chu kỳ | SẮP TỚI | Đỏ "Gần tới" |
| Dùng 85–95% chu kỳ | SẮP TỚI | Cam "Sắp tới" |
| Dùng dưới 85% chu kỳ | ĐỊNH KỲ | Xanh "Bình thường" |
