# MotoLog – V12: Đơn giản hoá tab trong Bảo dưỡng — bỏ "Đã hoàn thành", nối logic lọc thật

> Quyết định: bỏ tab "Đã hoàn thành" (đã có sẵn trong Lịch sử bảo dưỡng tổng thể, giữ lại sẽ trùng lặp). Chỉ còn 2 tab: **Tất cả** và **Sắp tới**. Đồng thời sửa lỗi: filter chip trước đây chỉ đổi màu khi bấm nhưng **không lọc list thật** — giờ nối logic lọc đúng, dùng lại cùng tiêu chí `urgency()` đã có ở `MaintenanceStore` (V6) để đồng nhất với màn Nhắc lịch, tránh trùng lặp cách tính.

---

## 1. Cập nhật `maintenance_screen.dart` — chỉ 2 tab, lọc thật

```dart
import 'package:flutter/material.dart';
import 'package:google_fonts/google_fonts.dart';
import '../theme/app_theme.dart';
import '../widgets/bottom_nav_bar.dart';
import '../widgets/quick_add_menu.dart';
import '../data/maintenance_store.dart';
import '../models/maintenance_item.dart';
import 'maintenance_item_detail_screen.dart';

class MaintenanceScreen extends StatefulWidget {
  const MaintenanceScreen({super.key});
  @override
  State<MaintenanceScreen> createState() => _MaintenanceScreenState();
}

class _MaintenanceScreenState extends State<MaintenanceScreen> {
  int _filterIndex = 0; // 0 = Tất cả, 1 = Sắp tới
  static const _filters = ['Tất cả', 'Sắp tới'];

  @override
  Widget build(BuildContext context) {
    return ListenableBuilder(
      listenable: MaintenanceStore.instance,
      builder: (context, _) {
        final store = MaintenanceStore.instance;
        final currentOdo = store.currentOdo;

        // ── Đây là phần LỌC THẬT, trước đây bị thiếu ──
        final List<MaintenanceItem> displayedItems = _filterIndex == 0
            ? store.items
            : store.items.where((i) => i.urgency(currentOdo) != 'normal').toList()
                ..sort((a, b) => a.remainingKm(currentOdo).compareTo(b.remainingKm(currentOdo)));

        return Scaffold(
          appBar: AppBar(title: const Text('Bảo dưỡng'), leading: const BackButton()),
          body: Column(children: [
            // Filter chips — chỉ 2 ô
            Container(
              color: Colors.white,
              padding: const EdgeInsets.symmetric(vertical: 10, horizontal: 16),
              child: Row(children: List.generate(_filters.length, (i) => GestureDetector(
                onTap: () => setState(() => _filterIndex = i),
                child: AnimatedContainer(
                  duration: const Duration(milliseconds: 200),
                  margin: const EdgeInsets.only(right: 8),
                  padding: const EdgeInsets.symmetric(horizontal: 18, vertical: 8),
                  decoration: BoxDecoration(
                    color: _filterIndex == i ? AppColors.primary : Colors.white,
                    borderRadius: BorderRadius.circular(20),
                    border: Border.all(color: _filterIndex == i ? AppColors.primary : AppColors.divider),
                  ),
                  child: Text(_filters[i], style: GoogleFonts.beVietnamPro(
                    fontSize: 13, fontWeight: FontWeight.w500,
                    color: _filterIndex == i ? Colors.white : AppColors.textSecondary,
                  )),
                ),
              ))),
            ),

            // List — dùng displayedItems đã lọc, có empty state riêng cho tab "Sắp tới"
            Expanded(
              child: displayedItems.isEmpty
                ? Center(
                    child: Text(
                      _filterIndex == 1 ? 'Không có hạng mục nào sắp tới hạn 🎉' : 'Chưa có hạng mục bảo dưỡng',
                      style: GoogleFonts.beVietnamPro(fontSize: 14, color: AppColors.textSecondary),
                    ),
                  )
                : ListView.separated(
                    padding: const EdgeInsets.symmetric(vertical: 12, horizontal: 16),
                    itemCount: displayedItems.length,
                    separatorBuilder: (_, __) => const SizedBox(height: 12),
                    itemBuilder: (_, i) {
                      final item = displayedItems[i];
                      final isOverdue = item.isOverdue(currentOdo);
                      final progress = item.progress(currentOdo);

                      return GestureDetector(
                        onTap: () => Navigator.push(context, MaterialPageRoute(
                          builder: (_) => MaintenanceItemDetailScreen(item: item),
                        )),
                        child: Container(
                          padding: const EdgeInsets.all(16),
                          decoration: BoxDecoration(
                            color: Colors.white,
                            borderRadius: BorderRadius.circular(14),
                            boxShadow: [BoxShadow(color: Colors.black.withOpacity(0.05), blurRadius: 8)],
                          ),
                          child: Row(children: [
                            Container(
                              width: 48, height: 48,
                              decoration: BoxDecoration(color: AppColors.greenChip, borderRadius: BorderRadius.circular(12)),
                              child: Icon(item.icon, color: AppColors.primary, size: 26),
                            ),
                            const SizedBox(width: 14),
                            Expanded(child: Column(crossAxisAlignment: CrossAxisAlignment.start, children: [
                              Text(item.name, style: GoogleFonts.beVietnamPro(fontSize: 15, fontWeight: FontWeight.w600)),
                              const SizedBox(height: 2),
                              Text('Định kỳ sau mỗi ${item.intervalKm} km', style: GoogleFonts.beVietnamPro(fontSize: 12, color: AppColors.textSecondary)),
                              const SizedBox(height: 8),
                              ClipRRect(
                                borderRadius: BorderRadius.circular(4),
                                child: LinearProgressIndicator(
                                  value: progress,
                                  backgroundColor: AppColors.greenChip,
                                  color: isOverdue ? const Color(0xFFD32F2F) : AppColors.primary,
                                  minHeight: 6,
                                ),
                              ),
                              const SizedBox(height: 4),
                              Text(
                                isOverdue ? 'Đã quá hạn ${item.overdueKm(currentOdo)} km' : 'Còn ${item.remainingKm(currentOdo)} km',
                                style: GoogleFonts.beVietnamPro(
                                  fontSize: 12, fontWeight: FontWeight.w600,
                                  color: isOverdue ? const Color(0xFFD32F2F) : AppColors.primary,
                                ),
                              ),
                            ])),
                          ]),
                        ),
                      );
                    },
                  ),
            ),

            // Nút "Thêm bảo dưỡng" — mở Quick Add Menu hoặc thẳng AddMaintenanceScreen tuỳ bạn muốn
            Padding(
              padding: const EdgeInsets.fromLTRB(16, 0, 16, 16),
              child: ElevatedButton.icon(
                onPressed: () {
                  // Có thể mở thẳng AddMaintenanceScreen() không preselect, để người dùng tự chọn hạng mục
                },
                icon: const Icon(Icons.add),
                label: const Text('Thêm bảo dưỡng'),
              ),
            ),
          ]),
          bottomNavigationBar: MotoBottomNavBar(
            currentIndex: 0,
            onTap: (i) {},
            onAddTap: () => showQuickAddMenu(context),
          ),
        );
      },
    );
  }
}
```

## 2. Vì sao bỏ "Đã hoàn thành" là đúng — tránh trùng lặp dữ liệu

| Nơi đã có dữ liệu "đã làm" | Vai trò |
|---|---|
| `MaintenanceItemDetailScreen` → mục "LỊCH SỬ BẢO DƯỠNG" (V11) | Lịch sử CHI TIẾT theo từng hạng mục riêng |
| `HistoryScreen` tab "Bảo dưỡng" (V4) | Lịch sử TỔNG HỢP tất cả hạng mục, theo thời gian |

→ Tab "Đã hoàn thành" trong `MaintenanceScreen` sẽ là **bản sao thứ 3** của cùng 1 dữ liệu, không cần thiết. Bỏ đi giúp giao diện gọn, đỡ rối, đúng theo nguyên tắc "1 nguồn dữ liệu, nhiều nơi xem" đã thống nhất từ V6.

## 3. Đồng nhất tiêu chí "Sắp tới" với màn Nhắc lịch — không tạo logic riêng

Tab "Sắp tới" ở đây dùng **chính xác cùng hàm** `item.urgency(currentOdo) != 'normal'` đã định nghĩa ở `MaintenanceItem` (V6) — đây cũng là tiêu chí mục "SẮP TỚI" trong `ReminderScreen` đang dùng. Vì vậy:

- Hạng mục nào hiện ở "Sắp tới" trong `MaintenanceScreen` → cũng sẽ xuất hiện ở mục "SẮP TỚI" của `ReminderScreen`, và ngược lại — **luôn khớp nhau, không lệch**.
- Không cần thêm field/cờ đánh dấu riêng nào khác — chỉ tính từ ODO hiện tại + dữ liệu hạng mục có sẵn.
